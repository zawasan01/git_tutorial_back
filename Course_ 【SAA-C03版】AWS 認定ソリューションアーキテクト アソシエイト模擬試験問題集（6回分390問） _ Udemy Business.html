ある会社は複数の企業と連携してAWS上に新規のアプリケーションを開発しています。パートナー企業が開発する機能と連携するためには、そのパートナー企業のAWSアカウントから、同社のAWSアカウントにあるAmazon SQSキューにアクセスして、キューをポーリングすることが必要です。

安全に権限を分割しつつ、パートナー企業にSQSキューへのアクセス権限を付与するには、どのようにすればよいでしょうか。

Explanation

オプション２が正解となります。異なるAWSアカウントに対してSQSキューへのアクセス権限を付与するためには、SQSアクセスポリシーを作成して、そのAWSアカウントを指定してキューへのメッセージ送受信を許可する設定を行います。


Amazon SQS キューにアクセスするには、SQS アクセスポリシー、IAM ポリシー、またはその両方にアクセス許可を設定する必要があります。別のアカウントへのアクセス許可を設定する場合は、SQS アクセスポリシーと IAM ポリシーの両方で、アクセスを許可するステートメントが必要です。

IAMポリシーでは、Amazon SQSを有するアカウント内でSQSメッセージの送信を許可する設定を行います。したがって、このポリシー自体では異なるAWSアカウントへのアクセス許可をしているわけではありません。具体的には以下のようになります。このポリシーでは、リソース arn:aws:sqs:us-east-1:123456789012:queue_1 にメッセージを送信するアクセス許可を username1 に付与します。


{

   "Version": "2012-10-17",

   "Statement": [{

      "Effect": "Allow",

      "Action": "sqs:SendMessage",

      "Resource": "arn:aws:sqs:us-east-1:123456789012:queue_1"

   }]

}


SQSアクセスポリシーでは、特定のアカウントを指定してメッセージの受信を許可することができます。以下のポリシーでは、username1 がキューにメッセージを送信することを許可しています。


{

   "Version": "2012-10-17",

   "Id": "Queue1_Policy",

   "Statement": [{

      "Sid":"Queue1_AllActions",

      "Effect": "Allow",

      "Principal": {

         "AWS": [

            "arn:aws:iam::111122223333:user/username1"

         ]

      },

      "Action": "sqs:SendMessage",

      "Resource": "arn:aws:sqs:us-east-1:123456789012:queue_1"

   }]

}



オプション１は不正解です。IAMポリシーによって設定するのは、同じアカウント内のAmazon SQSに対してメッセージを送信可能にするIAMポリシーであり、パートナ企業のアカウントに権限を付与するポリシーではありません。

オプション３は不正解です。インスタンスプロファイルを作成して、アクセス権限を付与することはできません。

オプション４は不正解です。IAMロールによる権限委譲では、そのAmazon SQSの設定操作自体を委譲する際などに利用します。IAMロールはSQSキューへのアクセス許可設定には利用しません。

大手出版社ではAmazon ECSを活用してマイクロサービス型の分散システムを構築しています。EC2インスタンスによる1つのマイクロサービスのプロセス処理が実施された後に、その結果を受けて別のマイクロサービスを構成するEC2インスタンスによる処理が実行されます。その際に処理イベントの発生タイミングに依存しない仕組みが必要です。

オプション４は不正解です。Lambdaイベントによって通知するといった処理はありません。

CloudFront配信時のコスト削減効果のある方法はどれでしょうか？

Explanation

オプション１が正解となります。オリジンからファイルを取得するときに、CloudFrontは各エッジロケーションでファイルを圧縮することができます。ビューワーがリクエストヘッダーに Accept-Encoding: gzip を含めるようのリクエストするとCloudFront はファイルを圧縮して、ファイルを供給するように設定できます。コンテンツが圧縮されるとファイルが小さくなるため、ダウンロード時間が短縮されます。これにより場合によっては、オリジナルの 4 分の 1 以下のサイズになることがあります。CloudFront のデータ転送コストは供給されたデータの総量に基づくので、圧縮ファイルを供給すると、非圧縮ファイルを供給する場合と比較して、通信コストが安くなります。

オプション３は不正解です。CloudFrontはLambda＠エッジではなく、エッジロケーションでファイルを圧縮します。 

オプション４は不正解です。S3による配信コンテンツの圧縮処理という機能はありません。

ある会社はストレージとしてAmazon EFSを利用して、複数のAmazon EC2インスタンスでファイルを共有する構成でアプリケーションの運用を開始したところです。最近になって、EFSファイルシステムのファイルデータが増加したことで、コストを削減するようにマネージャーから依頼されました。

どのようにしてコストを削減することができるでしょうか？

Explanation

Amazon EFS は標準ストレージクラスと低頻度アクセスストレージクラス (EFS IA) という２つのストレージクラスを利用しています。EFS IA は、毎日アクセスしないファイルに対して最適化されたコスト効率の料金/パフォーマンスを提供します。ファイルシステムは EFS ライフサイクル管理を有効にするだけで、選択したライフサイクルポリシーに従ってアクセスしないファイルは、自動的かつ透過的に EFS IA に移されます。EFS IA ストレージクラスの費用は、たったの 0.025 USD/GB-月* です。したがって、オプション１が正解となります。   



オプション２は不正解です。EFS IAを選択するのではなく、ライフサイクル管理を有効にして自動的にIAクラスへの移行するように設定することでEFS IAを利用します。 

オプション３は不正解です。EFS Standardストレージクラスは値段が高い方のクラスです。 

オプション４は不正解です。スループットモードは、ファイルシステムで実行できる全体のスループットを決定する際に利用する機能です。これを有効にするといった設定ではなく、バーストとプロビジョニングの 2 つのスループットモードから選択することになります。

ある会社はストレージとしてAmazon EFSを利用して、複数のAmazon EC2インスタンスでファイルを共有するアプリケーションの運用を開始したところです。このアプリケーションではファイルアクセス時に最小レイテンシーとなる構成が求められています。

この要件を満たすために、最適なソリューションを選択してください。

Explanation

オプション１が正解となります。Amazon EFS は、幅広いワークロードに対応するために必要なスループット、IOPS、および低レイテンシーを提供できるように設計されており、汎用と最大 I/O の 2 種類のパフォーマンスモードを提供します。

Amazon EFSの汎用モードは、ファイルシステムオペレーション単位で最小レイテンシーを実現するだけでなく、ランダムまたはシーケンシャル IO パターンでも同じ結果を得ることができます。したがって、最小レイテンシーを実現するためには汎用モードを選択することが必要です。 



オプション２は不正解です。Amazon EFSの最大 I/O モードのファイルシステムは、ファイルオペレーションのレイテンシーがわずかに長くなる代わりに、スループットが高性能にスケールできます。 

オプション３は不正解です。Amazon EFSのバーストスループットでは、スループットはファイルシステムのサイズに合わせてスケールされ、ファイルベースの多数のワークロードの不規則な性質に対応するために、必要に応じて動的にバーストされます。 

オプション４は不正解です。Amazon EFSのプロビジョニングされたスループットは、デフォルトのバーストモードより高い専用スループットを必要とするアプリケーションをサポートするように設計されており、ファイルシステムに格納されているデータの量と関係なく設定することができます。

ある会社は仮想サーバーを利用したオンプレミス環境にあるアプリケーションをAmazon EC2オンデマンドインスタンスを利用したAWS環境に移行したいと考えています。あなたはソリューションアーキテクトとして、移行テストを実施したところ、アプリケーション起動時に必要となるメモリの読み込みに、長時間かかっていることに気づきました。このアプリケーションの起動時間を短縮する必要があります。

この要件を満たすソリューションはどれでしょうか。

Explanation

休止の指示を受け取ったインスタンスは、ルートEBSボリュームに格納されたファイルにメモリ状態を書き出した上で、インスタンスをシャットダウンします。休止時にはインスタンスはメモリ状態を保持しているため、起動時にメモリの再読み込みにかかる時間を短縮することが可能となります。

オプション３は不正解です。オーケストレーション機能を有効化するだけでは、メモリの起動時間を短縮することはできません。

オプション４は不正解です。EC2スポットインスタンスを利用する要件がないため、今回のケースには不適切です。スポットインスタンスは、休止中の EC2 キャパシティーを使用するインスタンスで、オンデマンド価格より低料金で利用できます。中断される可能性があるため、常時実行されるようなアプリケーションのインスタンスには不適切です。

オプション２が正解となります。このシナリオでは、ECサイトのユーザーは世界中におり、すべてのユーザーがシステムを頻繁に使用しているわけではありません。したがって、一部リージョンで負荷が高くなっているため、Route53によるレイテンシールーティングを利用します。これにより、ユーザーによるリクエストをレイテンシーが低いリージョンにリダイレクトして、アプリケーションの処理を高速化します。


オプション１は不正解です。位置情報ルーティングはユーザーの位置情報に基づいてトラフィックをルーティングする場合に使用します。これは処理能力に基づいたルーティングではないため、不適切です。また、DynamoDBはデータ容量に対するオートスケーリングを有効化することなく、マネージド型サービスとして自動でデータ容量が拡張されます。 

オプション３は不正解です。CloudFrontを利用して、最適なリージョンに配信することは本件では求められていません。

オプション４は不正解です。Route53のフェールオーバールーティングはフェールオーバーによる可用性が高めることができますが、最適なパフォーマンスを達成するわけではありません。


Question 66:
Skipped

あなたの会社はビッグデータ解析を実施するWEBアプリケーションをホストするために、AWSとオンプレミス環境のハイブリッドクラウドインフラストラクチャを利用しています。保存データは滅多に利用されない中長期保存用となりますが、利用する場合は12時間以内にファイルを送信することが必要です。

これらの要件を満たす最も費用対効果の高いソリューションはどれでしょうか。

Explanation

このシナリオでは、データ分析の前後に発生する大量のデータを保存するためのコスト最適な方法が必要となっています。まずはデータをzipまたはtarファイルに圧縮することにより、データの保存コストを削減できます。保存データは12時間以内に取得できる必要があるため、Glacier よりもGlacier Deep Archiveに保存することでコストを抑えることができます。Glacier Deep Archiveはデータは ３ つ以上の AWS アベイラビリティゾーンにまたがって保存され、12 時間以内に取りだすことができます。したがって、オプション２が正解となります。    



オプション１と３は不正解です。Glacier Flexible Retrieval RetreivalやGlacier Instant RetrievalではなくGlacier Deep Archiveが最適なストレージタイプです。他のGlacierのストレージクラスではGlacier Deep Archiveよりもコストが高くなるため、コスト最適を達成できません。

オプション４は不正解です。S3 One-Zone-IAにGlacier ではなくGlacier Deep Archiveが最適なストレージタイプです。S3 One-Zone-IAではGlacier Deep Archiveよりもコストが高くなるため、コスト最適を達成できません。S3 One-Zone-IAは低頻度アクセス用のストレージではありますが、Glacierよりも取り出し頻度は高く、重要性の低いデータを保管するのに向いています。

Skipped

あなたの会社ではAWS Organizationsを利用したOU単位でのアクセス管理をSCPによって実施しています。デフォルトのSCP設定においてFullAWSAccessが設定されているOUに対して、DynamoDBの全権限を拒否する設定を追加しました。

この場合のOU内のメンバーアカウントの権限状況を説明してください。

Explanation

このシナリオでは、AWSアカウントにデフォルトで「FullAWSAccess」が付与されている場合に、どのように拒否形式（ブラックリスト形式）のSCPが機能するかが問われています。

オプション２が正解となります。SCPでば明示的な Deny 要素が、最初の明示的な Allow よりも優先されるため、影響を受けるアカウントのユーザーは DynamoDB アクションを実行できません。

デフォルトでは「FullAWSAccess」が付与されている場合、全リソースに対する全操作が明示的に許可されている状態となります。そこに、ブラックリスト形式で特定の操作を拒否すると、対象リソースの操作が拒否されるものの、その他のリソースについては「FullAWSAccess」が維持された状態となります。つまり、DynamoDB以外の操作が全て実施できることになります。


オプション１は不正解です。ブラックリスト形式で特定の操作を拒否すると対象リソースの操作が拒否されるものの、その他のリソースについては「FullAWSAccess」が維持された状態となります。したがって、DynamoDBの操作はできないため間違いです。

オプション３と４は不正解です。デフォルトのSCP「FullAWSAccess」を無効にするには、この設定を削除することが必要です。

ある会社は、Amazon EC2インスタンスを利用したアプリケーションを構築しています。EC2インスタンスは一般的なワークロードに対して利用され、その際に複数のインスタンス間での通信が多数発生します。このEC2インスタンスによるワーク処理を最適化することが求められています。

使用できる最も費用対効果の高いインスタンス構成は次のうちどれですか？

Explanation

新しい EC2インスタンスを起動する場合、EC2は相関性のエラーを最小限に抑えるために、すべてのインスタンスが基盤となるハードウェアに分散されるようにインスタンスを配置します。プレイスメントグループを使用することで、ワークロードのニーズに対応するために独立したインスタンスのグループのプレイスメントに影響を与えることができます。この設定は先にプレイスメントグループを構成した上で、その中でインスタンスタイプとインスタンス数を決定します。

オプション３は不正解です。プレイスメントグループの設定は先にプレイスメントグループを構成した上で、その中でインスタンスタイプとインスタンス数を設定することが必要です。

ある会社では、Amazon S3バケットを利用してユーザーに対する動画配信を行っています。現在、このS3バケットからコンテンツが配信されるユーザーのIPアドレス範囲が限定されていないためセキュリティ上問題となっています。そこで、特定のユーザーにのみ配信可能な方式に変更する必要があります。

オプション２は不正解です。S3にセキュリティグループを設定することはできません。 

オプション３は不正解です。CloudFrontにおける Referer制限はWAFを設定しないと実施できません。WAFのWEB ACLによる制御をReferer制限と呼びます。 

オプション４は不正解です。S3の事前署名付き URL を作成した場合、そのURLの有効期限内での特定プロジェクトの共有ができます。これはオブジェクトの制限付き共有に利用するものです。
